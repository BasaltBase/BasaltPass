# Glossary

Definitions of common terms used in BasaltPass documentation.

-   **Access Token**: A short-lived credential used to access protected resources.
-   **Authorization Code**: A temporary code exchanged for an Access Token.
-   **Client**: An application attempting to access a user's account.
-   **Grant Type**: The method used by a client to obtain an Access Token.
-   **IdP (Identity Provider)**: The service that authenticates users (BasaltPass).
-   **JWT (JSON Web Token)**: A compact, URL-safe means of representing claims to be transferred between two parties.
-   **OIDC (OpenID Connect)**: An identity layer on top of the OAuth 2.0 protocol.
-   **Resource Server**: The API that hosts the user's data.
-   **Tenant**: An isolated environment within the BasaltPass instance.
