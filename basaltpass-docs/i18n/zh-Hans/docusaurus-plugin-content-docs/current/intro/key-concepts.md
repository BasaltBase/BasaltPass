# 核心概念

理解这些概念对于有效使用 BasaltPass 至关重要。

## 租户 (Tenant)
租户是 BasaltPass 中的顶级容器。它代表一个组织、一个项目或一群用户。
-   **隔离性**: 租户 A 的用户无法看到租户 B 的数据。
-   **配置**: 每个租户可以有自己的登录策略、品牌和设置。

## 用户 (User)
属于特定租户的身份实体。
-   **属性**: 邮箱、密码哈希、个人资料数据。
-   **身份**: 用户通过 `sub` (subject) 声明在令牌中唯一标识。

## OAuth 客户端 (Client)
代表请求访问用户数据的应用程序（例如：Web 应用、移动 App、后端服务）。
-   **公共客户端**: 无法安全存储密钥的应用（SPA、移动端）。必须使用 PKCE。
-   **机密客户端**: 可以安全存储密钥的应用（传统 Web 应用、后端服务）。

## 角色与权限 (RBAC)
-   **权限**: 执行特定操作的能力（例如 `user.read`）。
-   **角色**: 一组权限的集合（例如 `Admin`, `Editor`）。
-   **分配**: 角色被分配给用户，授予他们拥有该角色的所有权限。
